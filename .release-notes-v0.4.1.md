v0.4.1 — Linux SymEngine + German Function Reference + deeper help

The release that completes the platform sweep: all five native
targets (iOS / macOS / Android arm64-v8a / Windows x86_64 / Linux
x86_64) now ship the SymEngine CAS. Plus a fully German Function
Reference and per-element help popovers across the module screens.

Headline changes since v0.4.0:

**SymEngine bridge — PLAN P11**

- **R130 — Linux x86_64 SHIPPED.** Bridge plugin now ships
  `libsymbolic_math_bridge.so` (18.3 MB stripped) at
  `linux/Libraries/`. CI at `.github/workflows/build-linux.yml`
  (vcpkg `x64-linux` static triplet on `ubuntu-22.04`, GLIBC 2.35
  baseline). Green on the first run — a hybrid of the Android
  (static-link the whole math stack) and Windows (three-mode
  consumer CMake) patterns, but simpler: a Linux ffiPlugin needs no
  registrar and the consumer path is pure bundling.
- **Windows loader fix (bridge 1.2.1).** The Dart loader now opens
  the bundled `libsymbolic_math_bridge.dll` first (where the wrapper
  symbols actually live in a consumer build), falling back to the
  registrar DLL. Fixes a likely runtime FFI failure on real Windows
  desktops. Non-regressive.
- Bridge pinned at `535ce5d` (v1.2.1).

**Function Reference i18n — R100**

- Complete **German** translation of the ~45-entry catalog
  (descriptions + every example hint) across all categories: CAS,
  number theory, high-precision, matrix/linear algebra, statistics,
  the constraints DSL, and Sudoku variants. Terminology follows
  German math didactics (Bildungspläne / de.wikipedia). French and
  Spanish gracefully fall back to English until translated.

**Deeper help — R105b (P6)**

- Per-element help popovers on the module-surface screens:
  Statistics test chips, Sudoku variant chips, and a help-mode
  Constraints-DSL operator reference row. Each opens the Function
  Reference popover (signature + localized description + "Learn
  more"), backed by a shared `showFunctionRefHelpPopover`.

**Tests:** 1992 → 2137 pass.

**Caveat:** build + link + symbol export are CI-verified on every
platform; full real-device runtime confirmation (Android / Windows /
Linux hardware) is still in progress.
