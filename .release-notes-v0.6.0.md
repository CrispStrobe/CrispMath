CrispCalc v0.6.0 — Deep CAS: ODEs, exact integration, series, step-by-step

The largest symbolic-math release yet. v0.6.0 takes CrispCalc's engine
from "strong" to genuinely deep: a full computer-algebra course of new
capabilities, every symbolic result checked against an independent
open-source CAS, and — for the classroom — step-by-step derivations.

Headline: a complete first-order + constant-coefficient second-order
ODE solver, exact rational-function integration, Taylor series, symbolic
linear systems, trigonometric-identity simplification, polynomial
inequalities, new 2D plot types, and multi-parameter / piecewise
user-defined functions. All of it works on the web too, and a silent
production outage that had quietly disabled the browser CAS is fixed.

────────────────────────────────────────────────────────────────────

## New symbolic capabilities

**Differential equations — `dsolve`.** The full first-order syllabus
plus constant-coefficient second order, exact and on every platform:

- Constant-coefficient linear (1st + 2nd order): `dsolve(y'' + 3y' + 2y = 0)`
  → `C1·e^(-x) + C2·e^(-2x)`; distinct/double/complex roots (exact
  surds), undetermined coefficients for polynomial / exponential /
  sinusoidal forcing with resonance handling.
- Separable: `dsolve(y' = x·y)` → `C1·e^(x²/2)`, incl. implicit forms.
- Linear integrating-factor: `dsolve(y' + 2y/x = x)` → `x²/4 + C1/x²`.
- Bernoulli: `dsolve(y' + y = y²)` → the explicit logistic
  `1/(C1·e^x + 1)`.
- Exact: `dsolve((2x + y) + (x + 2y)y' = 0)` → `x² + xy + y² = C1`.

**Exact integration of rational functions.** `integrate` is now a real
algorithm, not a rule-matcher: polynomial division, Yun squarefree
decomposition, Hermite reduction, and exact log/atan terms over linear
and quadratic factors — plus the Rothstein–Trager rational-root method,
which handles log-derivative numerators even over irreducible cubics:
`integrate((3x² + 1)/(x³ + x + 1), x)` → `log(x³ + x + 1)`.

**Taylor series & symbolic linear systems** (native SymEngine bindings):
`taylor(sin(x), x, 0, 7)`, `linsolve(x + y = 3; x - y = 1, x, y)`.

**Trigonometric-identity simplification:** `simplify(sin(x)² + cos(x)²)`
→ `1`, `simplify(2·sin(x)·cos(x))` → `sin(2x)` (Pythagorean, double
angle, power reduction, secant form).

**Polynomial inequalities:** `solve(x² − 4 > 0)` → `x < −2 ∨ x > 2`,
with unicode ≤/≥, exact surd endpoints, and the ≠ / point / ℝ / ∅ cases.

## Step-by-step ("show your work")

`dsolve` now produces a full derivation for the constant-coefficient
case — characteristic equation → roots (classified via the
discriminant) → homogeneous solution → particular → general — reachable
from the history row of any ODE result. The final line is taken
verbatim from the solver, so the shown work can never disagree with the
answer.

## Graphing

Three new 2D plot modes via a mode selector: **parametric** `(x(t),
y(t))`, **polar** `r(θ)`, and **implicit** `F(x, y) = 0` (rendered by
marching squares over the live view window).

## User-defined functions

- Multi-letter names and multiple parameters:
  `dist(a, b) = sqrt(a² + b²)`.
- **Piecewise bodies:** `f(x) = piecewise(x < 0, -x, x)`, evaluated at
  call time.

## Discoverability & languages

- New worked-examples for ODEs, series, linear systems, inequalities,
  the Rothstein–Trager integral, and piecewise — each localized in
  German, French, and Spanish.
- Function-reference examples for inequalities and rational integration;
  the German/French/Spanish `dsolve` descriptions now cover the full
  first-order syllabus.

## Web (browser build)

- **Fixed a silent production outage:** a startup-performance change had
  put `defer` on an inline loader script (where `defer` is ignored), so
  the in-browser SymEngine had been quietly failing to load and the web
  build had fallen back to a limited pure-Dart engine. The full CAS now
  runs in the browser again.
- All the new features — series, `linsolve`, trig simplify, ODEs,
  inequalities, integration, plots — work on the web. High-precision and
  number-theory functions were verified working in the browser too
  (they had been masked by the same loader bug).

## Correctness & testing

- **A SymPy-certified regression corpus** (108 cases): every expected
  symbolic result is cross-checked against an independent open-source
  CAS, run both headless (pure-Dart paths) and against the real native
  engine. It caught several real bugs on introduction, including
  near-integer display truncation and a limit engine blind to the
  bridge's complex-number formatting.
- The full test suite now stands at 4,000+ tests, green on all seven CI
  targets.

## Also in this release (pre-CAS fixes)

- `Ans` now chains at full precision: `8/3` then `Ans·3` shows `8`, not
  `8.00000000001`.
- DST-safe date arithmetic: `days between` no longer loses a day across
  a spring-forward.
- Scientific-notation display polish; Flutter 3.44 toolchain.

## Under the hood

- Native bridge updated to v1.5.0 (series / linsolve / trig simplify,
  with WASM exports for web parity).
- macOS: added the `network.client` sandbox entitlement (required for
  the AI verifier, cloud OCR, and crash reporting under the App Store
  sandbox). See `DISTRIBUTION.md` for the store-submission checklist.

## Downloads

**macOS** (`crisp_calc-…-macos.zip`) — full symbolic + numeric features.
Right-click → Open the first time (unsigned).

**iOS** (`crisp_calc-…-ios-unsigned.zip`) — full features. Inspection
only; sideloading needs a paid Apple Developer certificate.

**Linux** (`crisp_calc-…-linux-x64.tar.gz`) — full symbolic + numeric
features. SymEngine bridge ships as a bundled
`libsymbolic_math_bridge.so` (GLIBC 2.35 baseline; built on Ubuntu
22.04).

**Windows** (`crisp_calc-…-windows-x64.zip`) — full symbolic + numeric
features via the bundled `libsymbolic_math_bridge.dll`.

**Android** (`crisp_calc-…-android.apk`) — full features on arm64-v8a.
APK is unsigned; install with `adb install crisp_calc-….apk`.

All five native platforms ship the SymEngine CAS; the web build runs the
full CAS via WebAssembly. Build + link + symbol export are CI-verified
on each platform.
